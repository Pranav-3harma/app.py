About PranavCrypt

What is PranavCrypt?

PranavCrypt is a comprehensive web application that demonstrates four modern cryptographic algorithms: RSA, AES, ChaCha20, and ECC. Built with Flask and the Python cryptography library, it provides a secure, educational platform for exploring different encryption techniques.

RSA Algorithm

Type: Asymmetric Encryption

Key Size: 2048 bits

Use Case: Key exchange, digital signatures

Features:

  • Public key encryption, private key decryption
  • Hybrid encryption for large messages
  • OAEP padding for security
  • Mathematical foundation: Integer factorization
AES Algorithm

Type: Symmetric Encryption

Key Size: 256 bits

Mode: CBC (Cipher Block Chaining)

Features:

  • Block cipher with 128-bit blocks
  • Random IV generation
  • Industry standard encryption
  • High performance and security
ChaCha20 Algorithm

Type: Stream Cipher

Key Size: 256 bits

Nonce Size: 96 bits

Features:

  • High-performance stream cipher
  • Used in TLS 1.3 and other protocols
  • Constant-time implementation
  • Resistant to timing attacks
ECC Algorithm

Type: Elliptic Curve Cryptography

Curve: SECP256R1 (NIST P-256)

Use Case: Digital signatures

Features:

  • Smaller key sizes for same security
  • ECDSA digital signatures
  • SHA-256 hashing
  • Efficient and secure
Security Features
Data Protection
  • No persistent storage of keys or messages
  • All operations performed in memory
  • Automatic cleanup after operations
  • Secure session management
Cryptographic Security
  • Cryptographically secure random number generation
  • Proper key derivation and management
  • Input validation and sanitization
  • Secure padding and modes of operation
Technical Implementation
Backend Technology
  • Framework: Flask (Python)
  • Cryptography: Python cryptography library
  • Key Generation: os.urandom() for secure randomness
  • Encoding: Base64 for data transmission
Frontend Technology
  • Framework: Bootstrap 5
  • Icons: Font Awesome
  • JavaScript: Vanilla JS with Fetch API
  • Responsive: Mobile-friendly design
Usage Guidelines
Best Practices
  • Use this application for educational purposes only
  • Never use generated keys for production systems
  • Keep your private keys secure and never share them
  • Use strong, unique keys for each operation
  • Verify the integrity of encrypted data before decryption
Algorithm Selection Guide
  • RSA: Best for key exchange and small message encryption
  • AES: Ideal for bulk data encryption and general use
  • ChaCha20: Great for high-performance applications
  • ECC: Perfect for digital signatures and authentication
Important Disclaimer

This application is designed for educational and demonstration purposes only. While it implements proper cryptographic algorithms, it should not be used for securing sensitive production data. Always use established, audited cryptographic libraries and protocols for real-world applications.
